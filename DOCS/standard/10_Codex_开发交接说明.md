# 10 Codex 开发交接说明

## 1. 目标

本说明用于把项目直接交给 Codex 或自动化开发代理。  
要求其严格按标准文档实现，不自行切换技术栈，不自行扩展产品范围。

---

## 2. 必须遵守的基线

1. 使用 `FastAPI + PostgreSQL + Redis + Celery + Docker Compose`
2. 前端只对接内部 API，不直打官方接口
3. 只接官方开放接口，不做页面抓取
4. Token 必须单点刷新
5. 账户汇总接口必须支持计划聚合降级
6. 第一阶段只做阈值告警，不做定时简报
7. 第一阶段所有角色统一登录，不实现匿名公开页与大屏页
8. 其它服务器读取主服务最新 token，不自行刷新

---

## 3. 推荐开发顺序

1. 搭建 Web / Worker / Scheduler / PostgreSQL / Redis 基础骨架
2. 完成认证、审计、Token 管理
3. 完成官方集成与结构同步
4. 完成快照、聚合、账户汇总降级
5. 完成主看板与各榜单页面
6. 完成登录工作台、权限、归属与阈值告警
7. 完成部署、运维、验收

---

## 4. 输出要求

每一阶段都必须同时输出：

- 代码
- 数据迁移
- 可运行接口或页面
- 文档更新

禁止只留空壳模块。

---

## 5. 禁止事项

- 不要切回 Next.js / NestJS / Prisma / BullMQ
- 不要新增大屏页作为第一阶段正式需求
- 不要把 Token 暴露到普通前端接口
- 不要让页面直接请求官方 API
- 不要把阈值告警和定时简报混在一起实现
