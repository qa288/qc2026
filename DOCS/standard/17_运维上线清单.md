# 17 运维上线清单

版本：`v2.1`
状态：`生效中`
最后更新：`2026-03-30`
适用范围：发布、备份、值班、线上检查
读者对象：运维、后端、值班同事

## 1. 基线

- 域名：`qc.tyos.cc`
- 部署：Docker Compose
- 反代与站点管理：`1Panel`
- 路由：`/login`、`/`、`/api/*`，`/workbench` 仅保留兼容跳转
- 后台可公网访问，但必须登录

## 2. 服务清单

- web
- worker
- scheduler
- postgres
- redis

补充：

- 反向代理、证书和站点入口由 `1Panel` 托管
- 应用编排仍为 `Docker Compose`
- `1Panel` 不是应用编排真源，不能只在 `1Panel` 页面手工改容器参数

## 2A. 线上目录说明

- 当前线上目录：`/opt/qianchuan/qianchuan_openclaw_reporter`
- 当前线上目录不是 Git 工作树
- 正式代码真源是 GitHub 仓库

发布要求：

1. 同步代码文件到线上目录
2. 核对 `.env.dashboard`
3. 执行 Compose 重建
4. 验证 `healthz / readyz`

## 3. 持久化

必须持久化：

- `./volumes/postgres`
- `./volumes/redis`
- `./data`

说明：

- 容器日志当前由 Docker / `1Panel` 负责采集与查看，不单独维护日志卷目录。

## 4. Token

- 只允许主服务刷新 token
- PostgreSQL 为主真源
- `/app/data/qianchuan_latest_token.json` 为缓存副本
- 其它服务器只读 token-latest 或缓存文件

## 5. 通知

- 第一阶段先验证规则、记录和渠道配置结构
- 外发发送可在联调后按开关开启

## 6. 资源建议

- 试运行：`2C4G`
- 正式生产（1000 账户）：`8C16G` 起

## 7. 上线前检查

- 域名已解析
- HTTPS 生效
- `1Panel` 站点与反向代理配置已生效
- PostgreSQL / Redis 正常
- Web / Worker / Scheduler 正常
- OpenClaw 测试消息可达
- token 真源可持续刷新
- 备份策略已落地

备份要求：

- PostgreSQL 每日备份
- `./data` 每日归档
- Redis 仅做运行加速，不作为恢复真源
- 至少定期做一次恢复演练

## 8. 标准发布命令

```bash
cd /opt/qianchuan/qianchuan_openclaw_reporter
set -a && . ./.env.dashboard && set +a
docker-compose -f docker-compose.dashboard.yml up -d --build
```

发布后最少检查：

```bash
curl -s https://qc.tyos.cc/healthz
curl -s https://qc.tyos.cc/readyz
docker ps
```
