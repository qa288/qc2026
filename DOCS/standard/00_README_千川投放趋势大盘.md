## 1. 文档定位

本目录用于定义 **V1 正式开发基线**。
目录内如有冲突，按 [DOCS/README.md](../README.md) 中定义的文档优先级生效。

当前统一口径：

- 产品定位：内部经营与投流决策平台，所有角色均需登录
- 页面形态：`/login` 登录入口、`/` 登录后主工作台，`/workbench` 仅保留兼容跳转，不做大屏页、独立 `/admin`
- 官方数据：只走官方开放接口，不做页面抓取
- 通知范围：第一阶段只做 **阈值告警**
- 通知通道：第一阶段先把规则、通道配置和适配层做完整，实际对外推送可后置开启
- Token 策略：**单点刷新 + 共享最新 token**
- 账户汇总：官方接口异常时，允许 **计划聚合降级**
- 目标生产架构：**FastAPI + PostgreSQL + Redis + Celery + 独立 scheduler + Docker Compose**
- 当前正式域名：`qc.tyos.cc`

## 4. 当前标准范围

V1 标准范围包括：

- 登录与会话
- 角色与账户授权
- 总览页与内部工作台
- 账户、计划、商品/员工、素材榜单
- 时间段筛选、排序、搜索、详情侧栏
- 员工归属规则、关键词命中、人工修正
- 分钟级同步与历史保留
- 阈值告警
- 余额 / 共享钱包 / 爆单预警能力
- OpenClaw 多通道适配层
- Token 单点刷新与内部共享
- Docker Compose 部署

V1 不包括：

- 大屏模式
- 定时简报
- 浏览器抓取
- 多租户
- BI 设计器
- 单点登录

## 5. 核心原则

1. 只有一套主技术栈，不再保留 Node/Nest 方案作为正式要求。
2. 所有角色统一登录后进入工作台，不再以匿名公开页作为主形态。
3. 只有一套 Token 真源，不允许多实例各自刷新。
4. 阈值告警优先，定时简报后置。
5. 页面只查本地数据，不直接打官方接口。
6. 先把结构、同步、聚合、告警做稳，再做扩展功能。
