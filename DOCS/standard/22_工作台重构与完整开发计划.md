# 22 工作台重构与完整开发计划

最后更新：`2026-03-23`

本文档是当前阶段的总开发计划真源。  
后续页面、权限、上传、预警、数据同步和部署，都按本文档推进。

---

## 1. 当前最终口径

### 1.1 访问方式

- 所有角色都必须登录
- `/login` 为唯一登录入口
- `/` 为登录后的统一工作台首页
- `/workbench` 只保留兼容跳转，不再作为独立产品入口
- 不再以匿名公开页作为当前版本主形态

### 1.2 角色

- 管理员
  - 全量数据
  - 全量配置
- 主管
  - 仅看被分配账户范围内的全部数据
  - 如管理员开启“允许上传素材”，可在这些账户范围内创建素材上传任务
  - 不能配置运营关键词
- 运营
  - 不按账户范围直接看全部数据
  - 只看管理员分配给该运营账号关键词命中的数据

### 1.3 数据原则

- 页面查询只读 PostgreSQL / 缓存
- 官方接口只用于：
  - 定时同步
  - 历史缺口回补
  - 手动补数
  - token 换取与刷新
- 官方账户汇总异常时，允许计划聚合降级
- 能拉到的字段优先入库，再决定前端展示哪些字段

## 2. 总体技术架构

### 2.1 运行架构

- Web/API：FastAPI
- 主数据库：PostgreSQL
- 缓存/锁/队列：Redis
- 异步任务：Celery Worker
- 定时调度：独立 scheduler（Celery Beat）
- 部署：Docker Compose
- 反向代理与证书：1Panel

### 2.2 运行角色

- `web`
  - 页面和内部 API
- `worker`
  - 数据同步、历史回补、上传任务、预警计算
- `scheduler`
  - 发起周期任务
- `postgres`
  - 业务真源
- `redis`
  - 缓存、锁、任务队列

### 2.3 关键约束

- 页面不可直接调 OceanEngine 远程接口
- 手工同步必须异步入队，接口立即返回 `202`
- 热门查询可以缓存，但缓存不是数据真源
- 历史区间缺口必须先补库，再查库

## 3. 页面信息架构

### 3.1 管理员工作台

管理员主模块固定为：

1. 总览
2. 账户
3. 经营拆解
4. 计划
5. 素材
6. 运营账号排名
7. 账号与权限
8. 归属规则
9. 预警规则
10. 素材上传

### 3.2 主管工作台

主管沿用管理员工作台布局，但做两层裁剪：

- 数据范围裁剪到账户范围
- 配置能力裁剪

主管可见：

1. 总览
2. 账户
3. 经营拆解
4. 计划
5. 素材
6. 运营账号排名（仅账户范围内）
7. 预警规则（查看和使用，不做全局配置）
8. 素材上传（仅在开启上传权限后显示）

主管不可见：

- 账号与权限配置
- 运营关键词配置
- 全局归属规则维护

### 3.3 运营工作台

运营工作台单独收口，避免暴露复杂管理界面。

运营默认主模块：

1. 我的素材
2. 团队排名

顶部摘要：

- 今日总消耗
- 今日总支付
- 今日总订单
- 今日 ROI

- 新增账号

- 设置角色

- 配置主管账户范围

### 5.2 页面结构

分三层：

1. 账号列表
2. 账号编辑区
3. 角色扩展配置区

### 5.3 主管配置

- 账户范围
- 允许上传素材

### 5.4 运营配置

- 多个关键词
- 点击后查看当前关键词命中的素材列表
- 素材列表默认隐藏
- 仅在点中账号后展开

## 6. 素材上传模块

### 6.1 访问权限

可见角色：

- 管理员
- 已开启“允许上传素材”的主管

不可见角色：

- 运营

### 6.2 业务流程

1. 选择对象类型：
   - 账户
   - 计划
2. 输入关键词筛选
3. 展示命中结果
4. 若选账户：
   - 展开该账户下全部计划
   - 最终实际目标始终是计划
5. 勾选全部或部分计划
6. 选择 `1~50` 个本地视频文件
7. 创建上传任务
8. 后台异步：
   - 按账户分组上传
   - 逐计划绑定
9. 前端展示：
   - 上传进度
   - 绑定进度
   - 成功/失败明细

### 6.3 上传规则

- 同账户内按文件 hash 去重
- 页面只创建任务，不同步阻塞
- 主管只能上传到自己账户范围内的计划

## 8. 开发阶段

### Phase 1：基础架构与路由收口

- 取消匿名公开页
- `/` 改成登录后首页
- `/workbench` 改成兼容跳转
- 角色固定为管理员 / 主管 / 运营
- 用户表补齐角色和上传权限字段

### Phase 2：账号与权限模型

- 账号管理页重构
- 主管账户范围配置
- 主管上传权限配置
- 运营关键词配置
- session / API 返回统一角色能力

### Phase 3：管理员工作台重构

- 总览重构
- 账户 / 计划 / 素材 / 运营账号排名统一布局
- 所有模块统一时间维度和搜索交互
- 表格与详情面板去重复字段

### Phase 4：主管与运营工作台

- 主管：管理员工作台裁剪版
- 运营：我的素材 + 团队排名
- 运营首页顶部摘要

### Phase 5：素材上传

- 上传页 UI
- 上传目标搜索
- Celery 上传任务
- 进度轮询
- 失败重试

### Phase 6：预警中心

- 规则页面完成
- 渠道配置完成
- 记录与状态完成
- 余额 / 钱包 / 计划消耗 / 爆单规则打通

### Phase 7：收尾

- 全文档回写
- 服务器部署
- 仓库同步
- 验收清单回归

## 9. 当前开发要求

从本版开始，后续开发必须满足：

1. 先调架构和页面分层，再写具体功能
2. 页面不允许继续零碎修补式开发
3. 新能力必须先写进 `DOCS/standard/`
4. 文档、代码、部署结果必须保持一致
5. 每一阶段结束后都要回归时间维度、权限和数据来源
