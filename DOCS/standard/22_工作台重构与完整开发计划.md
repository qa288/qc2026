## 1. 当前最终口径

### 1.1 访问方式

- 所有角色都必须登录
- `/login` 为唯一登录入口
- `/` 为登录后的统一工作台首页
- `/workbench` 只保留兼容跳转，不再作为独立产品入口
- 不再以匿名公开页作为当前版本主形态

### 1.2 角色

- 管理员
  - 全量数据
  - 全量配置
- 主管
  - 仅看被分配账户范围内的全部数据
  - 如管理员开启“允许上传素材”，可在这些账户范围内创建素材上传任务
  - 不能配置运营关键词
- 运营
  - 不按账户范围直接看全部数据
  - 只看管理员分配给该运营账号关键词命中的数据

### 1.3 数据原则

- 页面查询只读 PostgreSQL / 缓存
- 官方接口只用于：
  - 定时同步
  - 历史缺口回补
  - 手动补数
  - token 换取与刷新
- 官方账户汇总异常时，允许计划聚合降级
- 能拉到的字段优先入库，再决定前端展示哪些字段

## 2. 总体技术架构

### 2.1 运行架构

- Web/API：FastAPI
- 主数据库：PostgreSQL
- 缓存/锁/队列：Redis
- 异步任务：Celery Worker
- 定时调度：独立 scheduler（Celery Beat）
- 部署：Docker Compose
- 反向代理与证书：1Panel

### 2.2 运行角色

- `web`
  - 页面和内部 API
- `worker`
  - 数据同步、历史回补、上传任务、预警计算
- `scheduler`
  - 发起周期任务
- `postgres`
  - 业务真源
- `redis`
  - 缓存、锁、任务队列

### 2.3 关键约束

- 页面不可直接调 OceanEngine 远程接口
- 手工同步必须异步入队，接口立即返回 `202`
- 热门查询可以缓存，但缓存不是数据真源
- 历史区间缺口必须先补库，再查库

## 3. 页面信息架构

### 3.1 管理员工作台

管理员主模块固定为：

1. 总览
2. 账户
3. 计划
4. 素材
5. 运营账号排名
6. 批量上传
7. 账号与权限
8. 预警规则

### 3.2 主管工作台

主管沿用管理员工作台布局，但做两层裁剪：

- 数据范围裁剪到账户范围
- 配置能力裁剪

主管可见：

1. 总览
2. 账户
3. 计划
4. 素材
5. 运营账号排名（仅账户范围内）
6. 批量上传（仅在开启上传权限后显示）
7. 预警规则（查看和使用，不做全局配置）

主管不可见：

- 账号与权限配置
- 运营关键词配置

### 3.3 运营工作台

运营工作台单独收口，避免暴露复杂管理界面。

运营默认主模块：

1. 我的素材
2. 团队排名

顶部摘要：

- 今日总消耗
- 今日总支付
- 今日总订单
- 今日 ROI

## 4. 管理员模块设计

### 4.1 总览

保留，但必须重构。

要求：

- 只保留高价值模块
- 不再堆叠过多卡片
- 去掉重复标题、重复说明、无意义大留白
- 核心指标一屏内完成判断

建议结构：

- 经营主卡
- 活跃度摘要
- 最近异常 / 重点关注
- 同步状态细栏

### 4.2 账户

字段：

- 账户名称
- 消耗
- 支付
- 订单
- ROI
- 账户状态

规则：

- 默认按消耗降序
- 所有字段可点表头升降序
- 支持：
  - 今日
  - 昨日
  - 近7天
  - 近30天
  - 指定日期范围
- 支持模块内关键词搜索

### 4.3 计划

字段：

- 计划名称
- 商品 / 主播
- 所属账户
- 订单数
- ROI
- 目标 ROI
- 消耗
- 支付
- 投放状态

规则：

- 消耗和 ROI 靠前
- 所有字段可点表头升降序
- 支持模块内关键词搜索
- 支持统一时间范围

### 4.4 素材

字段：

- 素材名称
- 消耗
- 订单数
- 支付
- ROI
- 计划数
- 账户数
- 播放预览入口

规则：

- 默认按消耗降序
- 所有字段可点表头升降序
- 支持模块内关键词搜索
- 支持统一时间范围
- 点击播放按钮后中间弹窗预览
- 若无可播放地址，则降级为封面/不可播放提示

### 4.5 运营账号排名

管理员看到的是“运营账号绑定关键词后的聚合排名”。

字段：

- 运营账号 / 展示名
- 消耗
- 订单数
- 支付
- ROI
- 计划数
- 账户数

规则：

- 默认按消耗降序
- 所有字段可点表头升降序
- 支持统一时间范围
- 支持模块内关键词搜索

## 6. 开发阶段

### Phase 1：基础架构与路由收口

- 取消匿名公开页
- `/` 改成登录后首页
- `/workbench` 改成兼容跳转
- 角色固定为管理员 / 主管 / 运营
- 用户表补齐角色和上传权限字段

### Phase 2：账号与权限模型

- 账号管理页重构
- 主管账户范围配置
- 主管上传权限配置
- 运营关键词配置
- session / API 返回统一角色能力

### Phase 3：管理员工作台重构

- 总览重构
- 账户 / 计划 / 素材 / 运营账号排名统一布局
- 所有模块统一时间维度和搜索交互
- 表格与详情面板去重复字段

### Phase 4：主管与运营工作台

- 主管：管理员工作台裁剪版
- 运营：我的素材 + 团队排名
- 运营首页顶部摘要

### Phase 5：素材上传

- 上传页 UI
- 上传目标搜索
- Celery 上传任务
- 进度轮询
- 失败重试

### Phase 6：预警中心

- 规则页面完成
- 渠道配置完成
- 记录与状态完成
- 余额 / 钱包 / 计划消耗 / 爆单规则打通

### Phase 7：收尾

- 全文档回写
- 服务器部署
- 仓库同步
- 验收清单回归

## 9. 当前开发要求

从本版开始，后续开发必须满足：

1. 先调架构和页面分层，再写具体功能
2. 页面不允许继续零碎修补式开发
3. 新能力必须先写进 `DOCS/standard/`
4. 文档、代码、部署结果必须保持一致
5. 每一阶段结束后都要回归时间维度、权限和数据来源
