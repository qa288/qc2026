# 05 内部接口与集成规范

## 1. 设计原则

1. 前端只调用内部 API，不直接调用官方接口。
2. 所有页面数据先落本地，再由内部 API 对外提供。
3. 只保留一套内部 API 命名与返回口径。
4. 接口文档必须区分：已实现、V1 必做、后续规划。
5. Token 接口只用于内部服务协同，不对普通前端暴露。
6. 页面查询请求不得直接打官方远程报表；若缺历史数据，应先补库，再继续读库。

## 3. 统一响应格式

```json
{
  "code": 0,
  "message": "OK",
  "data": {},
  "requestId": "req_xxx"
}
```

错误示例：

```json
{
  "code": 4001,
  "message": "BAD_REQUEST",
  "data": null,
  "requestId": "req_xxx"
}
```

## 6. 时间范围参数

统一支持两种方式：

### 6.1 预定义

- `range=day`
- `range=yesterday`
- `range=week`
- `range=month`

### 6.2 自定义

- `range=custom`
- `start_date=YYYY-MM-DD`
- `end_date=YYYY-MM-DD`

规则：

- 单次跨度不得超过 180 天
- 开始日期不得晚于结束日期

补充规则：

- 若查询范围内缺历史日快照，允许服务端先按天回补再返回
- 回补完成后，后续请求必须继续走本地库，不得每次重复远程拉取

## 7. Token 内部接口规则

### `GET /api/system/integrations/ocean-engine/token-latest`

用途：

- 给其它服务器读取最新 token

返回：

- `access_token`
- `refresh_token`
- `expires_at`
- `updated_at`
- `app_id`
- `customer_center_id`

要求：

- 只允许内部可信服务调用
- 不进入普通业务页面接口列表
- PostgreSQL `oauth_tokens` 为逻辑真源
- `/app/data/qianchuan_latest_token.json` 为缓存副本
- token 刷新必须使用 Redis 锁，禁止多实例并发刷新同一套 `refresh_token`

## 8. 官方接口适配规则

官方接口统一经 `integration service` 适配：

- Token
- 子账户
- 计划
- 计划详情
- 商品
- 素材
- 自定义主题数据

适配层职责：

- 重试
- request_id 记录
- 错误码映射
- 字段口径统一
- 账户汇总降级
- 历史日快照回补
- 重查询结果预聚合

---

## 9. 特殊规则

### 9.1 账户汇总

- 优先使用官方账户汇总接口
- 异常时自动回退到计划聚合
- 响应必须带来源标记

### 9.2 空值

- 不适用字段统一返回 `null`
- 前端显示 `--`

### 9.3 手工同步接口

- `POST /api/sync`
- `POST /api/sync/extended`

当前正式规则：

- 只负责把任务提交给 Celery
- 接口立即返回 `202`
- 不阻塞等待同步完成

### 9.4 素材榜查询

当前正式规则：

- 优先读 `material_rollups`
- 若历史老快照还没有 `material_rollups`，才回退到 `material_snapshots` 聚合
- 回退结果应尽快在后续同步中补成预聚合表
