# 20 角色模型与运营工作台方案

最后更新：`2026-03-23`

本文档用于替换此前“匿名公开页 + 管理后台”的旧口径，统一定义新的角色模型、访问方式、数据范围和运营工作台能力。

---

## 1. 当前最新决策

从本版开始，访问模型统一为：

- 所有角色都必须登录
- 不再保留匿名公开页作为长期主形态
- 首页改为登录后首页

当前角色统一分为 3 类：

1. `管理员`
2. `主管`
3. `运营`

这 3 类角色共享一套系统，但：

- 可见数据范围不同
- 可操作页面不同
- 可修改配置不同

---

## 2. 角色定义

### 2.1 管理员

管理员拥有全局权限：

- 可查看所有账户
- 可查看所有计划
- 可查看所有商品
- 可查看所有素材
- 可查看全部团队排名
- 可配置角色、账户范围、关键词绑定、人工修正
- 可创建上传任务
- 可管理预警规则
- 可管理系统级配置

管理员是唯一可以配置：

- 主管的账户范围
- 运营账号的关键词集合

### 2.2 主管

主管是“账户范围受限”的管理者：

- 可查看被分配账户范围内的全部数据
- 可查看这些账户下的所有计划、商品、素材
- 可查看这些账户范围内的团队排名
- 可查看这些账户范围内的预警与任务

主管和管理员的主要区别不是页面，而是**数据范围**：

- 管理员：所有账户
- 主管：部分账户

### 2.3 运营

运营不是按账户范围直接看全部数据，而是按“管理员分配给该运营账号的关键词”来看自己的内容。

运营能看到的数据，来自：

- 管理员在后台配置的“运营账号 -> 关键词”

示例：

- 运营账号：`xiaoyu_operator`
- 关键词：
  - `小宇`
  - `小宇剪辑`

则后续只要：

- 素材名命中 `小宇`
- 或计划名命中 `小宇`
- 或商品名命中 `小宇`
- 或账户维度归属于 `小宇`

这些数据都进入该运营的可见范围与团队统计。

---

## 3. 关键词与运营归属规则

### 3.1 绑定模型

管理员可以对某个运营账号配置一个或多个关键词。

例如：

- 运营：`小宇`
- 关键词：
  - `小宇`
  - `小宇剪辑`
  - `XY`

这表示：

- 这些关键词是该运营账号的识别规则
- 不是简单搜索词，而是“运营归属规则”

### 3.2 命中范围

关键词绑定到运营账号后，系统按以下对象去命中：

1. 素材名
2. 计划名
3. 商品名
4. 账户名

推荐默认优先级：

1. 素材命中
2. 计划命中
3. 商品命中
4. 账户命中

### 3.3 归属结果

命中后，数据会统一归到：

- 该关键词
- 该关键词所属运营账号
- 命中的账户 / 计划 / 商品 / 素材对象

所以运营看到的不是“平台原生员工数据”，而是：

- 基于关键词规则派生出来的运营归属数据

### 3.4 多对多规则

当前默认规则：

- 一个运营账号可以对应多个关键词
- 一个关键词可以分配给多个运营账号
- 一个关键词也可能命中多个账户

例如：

- 运营 `小宇` 可有关键词：
  - `小宇`
  - `小宇剪辑`
- 运营 `小刘` 可有关键词：
  - `小刘`
- 某个账户名里包含 `小宇`
- 另一个账户名里也包含 `小宇`

则：

- 两个账户都可以被 `小宇` 命中
- 同一个关键词不要求唯一绑定到单个账户

### 3.5 不再单独配置“账户关键词”

账户本身不需要额外单独维护关键词字段。

原因：

- 账户自带账户名称
- 搜索时本来就会按账户名称去命中关键词

因此 V1 不额外做：

- “账户 -> 关键词” 独立配置表

V1 只做：

- “运营账号 -> 多个关键词”

## 4. 首页与页面结构调整

由于所有角色都需要登录，因此页面结构调整为：

- `/login`
- `/`
- `/api/*`

说明：

- `/` 不再是匿名公开页
- `/` 改成登录后的首页
- 首页标题可继续使用：`基础数据排行`

---

## 5. 各角色首页能力

### 5.1 管理员首页

管理员首页继续保留完整工作台：

- 总览
- 账户
- 经营拆解
- 计划
- 素材
- 运营账号排名
- 归属规则
- 权限
- 预警规则
- 素材上传

管理员重点查看 4 类核心数据视图：

1. 账户
2. 计划
3. 素材
4. 运营账号排名

#### 账户视图

管理员可看到全量账户数据，至少包含：

1. 账户名称
2. 消耗
3. 支付
4. 订单
5. ROI
6. 账户状态

默认排序：

- 消耗降序

要求：

- 所有字段可点击表头切换升序 / 降序
- 支持统一时间范围：
  - 今日
  - 昨日
  - 近7天
  - 近30天
  - 指定日期范围
- 支持模块内关键词搜索
- 搜索规则：
  - 在当前账户表结果集内做全局包含匹配
  - 只要与关键词相关的账户数据都要显示出来

#### 计划视图

管理员可看到全量计划数据，至少包含：

1. 计划名称
2. 商品 / 主播
3. 所属账户
4. 订单数
5. ROI
6. 目标 ROI
7. 消耗
8. 支付
9. 投放状态

字段顺序要求：

- 消耗和 ROI 靠前

要求：

- 所有字段可点击表头切换升序 / 降序
- 支持统一时间范围：
  - 今日
  - 昨日
  - 近7天
  - 近30天
  - 指定日期范围
- 支持模块内关键词搜索
- 搜索规则：
  - 在当前计划表结果集内做全局包含匹配
  - 可命中计划名、商品、主播、账户等表格内相关字段

#### 素材视图

管理员可看到全量素材数据，至少包含：

1. 素材名称
2. 消耗
3. 订单数
4. 支付
5. ROI
6. 计划数
7. 账户数

默认排序：

- 消耗降序

要求：

- 所有字段可点击表头切换升序 / 降序
- 支持统一时间范围：
  - 今日
  - 昨日
  - 近7天
  - 近30天
  - 指定日期范围
- 支持模块内关键词搜索
- 搜索规则：
  - 在当前素材表结果集内做全局包含匹配
  - 可命中素材名、账户名、计划名等表格内相关字段

#### 运营账号排名视图

管理员可看到运营账号对应的关键词归属排名。

这里的“运营账号排名”本质上是：

- 运营账号绑定的关键词
- 命中到账户 / 计划 / 商品 / 素材后的聚合结果

至少包含：

1. 运营账号 / 展示名
2. 消耗
3. 订单数
4. 支付
5. ROI
6. 计划数
7. 账户数

默认排序：

- 消耗降序

要求：

- 所有字段可点击表头切换升序 / 降序
- 支持统一时间范围：
  - 今日
  - 昨日
  - 近7天
  - 近30天
  - 指定日期范围
- 支持模块内关键词搜索
- 搜索规则：
  - 在当前运营账号排名结果集内做全局包含匹配
  - 可命中运营账号名、展示名以及相关聚合对象名称

### 5.2 主管首页

主管看到的页面结构可与管理员一致，但数据自动裁剪到账户范围。

主管不允许：

- 修改全局角色
- 修改系统级配置
- 修改运营账号的关键词集合

### 5.3 运营首页

运营首页应裁成两个主模块：

1. `我的素材`
2. `团队排名`

不要把管理员和主管的复杂工作台原样暴露给运营。

## 6. 运营页设计

### 6.1 我的素材

运营可以看到自己归属范围内的素材列表。

字段要求：

1. 素材名称
2. 消耗
3. 归属账户
4. 归属计划
5. 播放入口

时间筛选统一支持：

- 今日
- 昨日
- 近7天
- 近30天
- 指定日期范围

默认：

- 今日

排序：

- 至少支持 `消耗` 升序 / 降序
- 后续可继续扩到 ROI、订单

### 6.2 团队排名

每个运营都可以看到团队排名。

团队排名展示的是：

- 各运营关键词归属组
- 在指定时间范围内的消耗排名

字段建议：

1. 排名
2. 归属人
3. 消耗
4. 账户数
5. 计划数

支持：

- 今日
- 昨日
- 近7天
- 近30天
- 指定日期范围

支持：

- 消耗升序 / 降序

## 7. 素材播放能力结论

### 7.1 当前建议

V1 不把“官方一定返回可直接播放链接”当成前提。

当前系统里已经能稳定拿到的素材相关字段主要包括：

- `material_id`
- `material_name`
- `video_id`
- `image_url`
- `cover_url`

其中：

- `cover_url / image_url` 更适合作为缩略图或预览封面
- “直接可播放的公开视频 URL” 当前不应在 V1 中作为强依赖

### 7.2 V1 建议实现

运营页第五列先做成：

- `预览`

形式建议：

- 有 `cover_url` 时显示封面缩略图
- 点击后弹出预览弹层
- 如果后续联调确认官方返回了稳定可播放地址，再接入视频播放

### 7.3 不建议的承诺

当前不建议直接向业务承诺：

- 所有素材都能在系统里原地播放

因为这依赖官方素材返回字段、鉴权、直链稳定性。

V1 更稳的策略是：

- 先支持封面预览
- 可播放链路作为增强项

## 8. 数据权限模型

### 8.1 管理员

- 全量数据

### 8.2 主管

- 账户范围过滤
- 计划 / 商品 / 素材 / 预警 / 上传任务全部继承账户范围

### 8.3 运营

运营的可见范围只取决于：

- 数据对象是否命中该运营账号的关键词规则

因此运营不需要单独再做账户授权。

运营可见数据集的条件是：

1. 数据已进入管理端底库
2. 数据对象命中分配给该运营的关键词规则

## 9. 数据模型新增建议

### 9.1 `operator_keywords`

管理员把关键词绑定到运营账号：

- `id`
- `operator_user_id`
- `keyword`
- `enabled`
- `priority`
- `created_at`

### 9.2 `operator_material_views`

可选预聚合表：

- `snapshot_date`
- `operator_user_id`
- `advertiser_id`
- `ad_id`
- `material_id`
- `material_name`
- `cover_url`
- `stat_cost`

如果不单独建表，也可以由现有素材快照 + 关键词绑定规则动态聚合。

V1 建议：

- 先动态聚合
- 后续再做预聚合优化

## 10. 架构调整点

在现有架构基础上，需要新增一个“角色分视图层”：

- 管理员：完整工作台
- 主管：完整工作台但数据裁剪
- 运营：简化工作台

这不是新增技术栈，而是新增：

- 登录后首页路由分发
- 后端权限裁剪
- 前端按角色渲染导航和视图

### 10.1 不需要推翻的部分

以下架构继续保留：

- FastAPI
- PostgreSQL
- Redis
- Celery
- Scheduler
- Docker Compose

### 10.2 需要新增的部分

- 角色首页分发逻辑
- 运营账号关键词表
- 运营素材列表接口
- 团队排名接口按角色裁剪

## 11. V1 开发顺序

### Phase 1

- 角色模型调整
- 去掉匿名公开页前提
- 登录后首页分发
- 管理员 / 主管 / 运营导航区分

### Phase 2

- 运营账号关键词配置
- 关键词命中到账户 / 计划 / 商品 / 素材
- 运营我的素材页
- 运营团队排名页

### Phase 3

- 运营素材预览
- 排序增强
- 预聚合优化

---

## 12. 当前默认约定

若没有新的补充说明，默认按以下规则开发：

1. 所有角色都必须登录
2. 角色分为：管理员 / 主管 / 运营
3. 管理员全量
4. 主管按账户范围
5. 运营按“管理员分配给该运营账号的关键词”看数据
6. 运营页面默认只有：
   - 我的素材
   - 团队排名
7. V1 素材预览先做封面/预览，不强依赖可播放直链

---

## 13. 后续补需求的记录方式

如果后续你继续补角色或运营需求，统一按下面结构追加：

1. 角色
2. 页面
3. 可见数据范围
4. 关键字段
5. 排序规则
6. 时间维度
7. 是否可操作
8. 是否可导出

这样可以避免角色需求继续碎片化。
