# 21 管理员预警规则页方案

## 1. 目标

本文是预警规则模块的唯一专题真源。  
其它文档只允许引用，不再重复展开规则类型、渠道和页面结构。

管理员侧增加独立的“预警规则”页面，用于配置和查看：

- 单账户余额预警
- 共享钱包余额预警
- 计划消耗预警
- 爆单预警

页面重点不是技术能力展示，而是让不懂技术的运营或管理人员也能顺着页面完成：

1. 选择预警类型
2. 搜索并选择监控对象
3. 设置阈值和冷却时间
4. 选择是否外发通知
5. 查看已生效规则和触发记录

---

## 2. 页面定位

该页面仅对管理员开放。

主管：

- 当前版本不配置运营关键词
- 当前版本也不配置预警规则

运营：

- 不进入预警规则页
- 只查看与自己相关的数据结果

---

## 3. 第一阶段支持的规则类型

### 3.1 已实现并可实际触发

- 账户余额预警
- 共享钱包余额预警
- 计划消耗预警
- 计划爆单预警

### 3.2 已预留但暂不作为已实现能力承诺

- 商品爆单预警

说明：

- 页面和文档可以预留“商品爆单”位置
- 但在后端完成独立触发计算前，不应作为已生效能力对外承诺

---

## 4. 目标搜索规则

预警规则页的“限定对象”必须支持关键词搜索。

### 4.1 搜索原则

- 输入关键词
- 下拉候选项按当前规则类型过滤
- 用户从候选项中选择具体对象
- 前端展示人类可读名称
- 后端保存真实 ID

不允许：

- 让用户直接输入对象 ID
- 让用户只看到纯 ID 值

### 4.2 按规则类型对应的搜索范围

账户余额预警：

- 搜账户名称
- 选择账户

共享钱包余额预警：

- 搜共享钱包名称
- 选择共享钱包

计划消耗预警：

- 搜计划名称
- 选择计划

计划爆单预警：

- 搜计划名称
- 选择计划

商品爆单预留：

- 后续搜商品名称
- 选择商品

### 4.3 留空逻辑

- 如果用户不选择具体对象
- 则该规则对“当前类型下全部对象”生效

---

## 5. 通知渠道设计

第一阶段页面只保留这三种通知渠道：

- 飞书
- 钉钉
- 微信

说明：

- 当前先把配置结构、页面和参数保留完整
- 实际底层发送适配可以分阶段打开
- 不再在第一阶段页面中暴露 Slack、Telegram、Email 等低优先级渠道

### 5.1 页面字段

- 是否开启通知
- 通知渠道
- 通知账号
- 接收目标
- 是否启用阈值外发
- 单次发送条数

### 5.2 用户理解方式

页面文案要用业务语言：

- “通知渠道”
- “接收目标”
- “阈值告警”

避免技术型表述：

- `target_id`
- `open_id`
- `chat_id`

如确需填写技术标识，放到占位说明里，不作为标题文案。

---

## 6. 页面结构

预警规则页固定为三层结构：

### 6.1 顶部摘要

- 通知状态
- 已启用规则数
- 余额 / 钱包规则数
- 爆单规则数

### 6.2 中间配置区

左侧：规则编辑器

- 模板按钮
- 规则类型
- 指标
- 搜索并选择对象
- 比较符
- 阈值
- 冷却分钟
- 最低消耗
- 备注
- 启用开关

右侧：通知渠道

- 是否开启
- 渠道
- 接收目标
- 账号
- 批量发送条数

### 6.3 底部结果区

- 已生效规则表
- 后续可扩触发记录表

---

## 7. 交互要求

### 7.1 模板优先

预警规则页提供模板按钮：

- 账户余额
- 共享钱包
- 计划消耗
- 爆单计划

点击模板后：

- 自动填好对象类型
- 自动填好指标
- 自动填好比较关系
- 自动填好默认阈值
- 用户只需要再补充限定对象和备注

### 7.2 搜索并选择对象

- 文本框显示“可搜索的人类名称”
- 隐藏字段保存真实 ID
- 如果输入了关键词但没有从候选项中选择具体对象
- 提交时必须阻止并提示

### 7.3 无需技术理解

页面操作顺序要让非技术人员能顺着走：

1. 点模板
2. 搜对象
3. 填阈值
4. 选是否外发
5. 保存

---

## 8. 数据与实现要求

### 8.1 当前真实数据来源

账户余额：

- `account_balances`

共享钱包：

- `shared_wallets`
- `shared_wallet_account_relations`

计划消耗 / 计划爆单：

- `plan_snapshots`

### 8.2 当前真实后端能力

当前已实现的规则实体：

- `account_balance`
- `shared_wallet`
- `plan`
- `burst_plan`

当前已实现的规则指标：

- `account_balance`
- `wallet_balance`
- `stat_cost`
- `burst_order_count`
- 以及账户 / 计划通用指标

### 8.3 后续扩展方向

后续新增“商品爆单”时：

- 新增 `burst_product`
- 计算逻辑基于 `product_snapshots`
- 前端规则模板与搜索范围同步扩展

---

## 9. 验收标准

第一阶段验收通过标准：

- 管理员能进入预警规则页
- 能快速创建账户余额规则
- 能快速创建共享钱包规则
- 能快速创建计划消耗规则
- 能快速创建计划爆单规则
- 能按关键词搜索对象并选中
- 规则保存后可在列表中看到
- 通知渠道可切换到飞书 / 钉钉 / 微信
- 通知未启用时，不影响规则本身的页面内记录

---

## 10. 不做的事情

第一阶段不做：

- 商品爆单已实现承诺
- 多渠道联动策略
- 分级升级通知
- 复杂编排器
- 面向技术人员的高级脚本配置
