# 19 素材上传与批量投放方案

最后更新：`2026-03-23`

本文档用于记录“按关键词筛选账户/计划，然后批量上传视频素材到所选计划”的产品要求、架构调整点、数据模型、任务拆分和开发顺序。

本文是素材上传模块的唯一专题真源。  
其它文档只允许引用，不再重复展开上传细节。

这是一个增量需求文档。后续如果继续补“图片上传”“素材替换”“素材模板”“多素材包投放”等能力，统一追加到本文档，不再另起平行方案。

## 当前实现进度

截至 `2026-03-24`，代码里已经落地的是：

- 管理员与已授权主管可见“上传素材”入口
- 支持按 `账户 / 计划` 搜索目标
- 账户模式会展开命中账户下的全部计划
- 支持勾选计划、选择本地多个视频文件、创建上传任务
- 上传任务会把文件先存到本地任务目录，并写入 PostgreSQL 任务表
- 已接通后台异步上传执行器：
  - 按账户分组处理
  - 同账户按文件 hash 复用已上传素材
  - 逐计划绑定素材
- 任务状态、文件数、计划数和失败原因已可回显

当前剩余未完全落地的是：

- 更细的字节级实时上传百分比
- 图片素材上传
- 替换已有素材的高级流程

当前线上已知限制：

- 若广告主没有官方 `/open_api/v3.0/local/file/video/upload/` 权限，上传任务会失败
- 这属于广告主权限问题，不是当前执行架构问题

## 1. 需求背景

当前系统已经具备：

- 账户 / 计划 / 商品 / 素材 / 视频的查询与落库
- 关键词匹配账户 / 计划 / 商品 / 素材
- Celery 异步任务体系
- PostgreSQL / Redis / Docker Compose 的正式运行架构

当前新增需求是：

- 后台支持通过关键词搜索账户
- 展开这些账户下的全部计划
- 支持批量勾选全部或部分计划
- 从本地选择一个或多个视频素材文件
- 将这些视频素材批量上传到所选计划
- 上传过程必须异步执行，并有进度反馈
- 素材上传可开放给主管，但必须由管理员单独授权

---

## 2. 业务规则

### 2.0 可访问角色

V1 上传页允许这两类角色进入：

- 管理员
- 已开启“允许上传素材”的主管

运营账号不能进入素材上传页。

主管的上传目标受账户范围限制：

- 只能搜索和选择自己账户范围内的账户
- 只能展开这些账户下的计划
- 不能越过账户范围上传到其他账户

### 2.1 基本选择流程

上传页面的目标选择按下面顺序执行：

1. 选择对象类型：
   - `账户`
   - `计划`
2. 输入关键词：
   - 可为空
   - 为空表示当前对象类型下不过滤关键词
3. 展示命中对象
4. 若当前对象类型为 `账户`：
   - 先列出匹配到账户
   - 再自动展开这些账户下的全部计划
   - 最终实际投放目标始终是“计划”
5. 若当前对象类型为 `计划`：
   - 直接列出命中的计划
6. 支持：
   - 全选
   - 取消全选
   - 部分勾选
7. 选择本地视频文件：
   - 单次允许 `1 ~ 50` 个文件
8. 点击上传后：
   - 创建异步任务
   - 后台逐账户、逐计划执行
   - 前端展示进度和结果

### 2.2 关键词命中规则

搜索范围与已有归属规则体系保持一致：

- 账户模式：只搜账户名
- 计划模式：只搜计划名
- 关键词规则为“不区分大小写的包含匹配”

注意：

- 本功能里的关键词搜索是“选择投放目标”的筛选器
- 不是员工归属规则
- 两者可以复用同一套搜索实现，但不能混成同一个业务概念

### 2.3 目标对象规则

V1 只支持“上传到计划”，不支持“直接上传到账户但不落计划”。

原因：

- 账户只是筛选入口
- 真正要投放的对象是计划

因此：

- 选择账户 = 先选账户，再展开到账户下全部计划
- 选择计划 = 直接对这些计划上传

### 2.4 素材类型范围

V1 只支持：

- 视频素材

V1 不做：

- 图片素材
- 图文素材
- 标题素材
- 素材替换
- 素材删除

这些能力后续如果要做，另作为 V2/V3 增量补充。

## 3. 推荐实现策略

### 3.1 结论

主链路采用：

- 前端：只创建任务、轮询进度、展示结果
- 后端：Celery 异步执行
- 官方接口：逐账户上传，逐计划绑定

不采用：

- 页面点击后同步阻塞上传
- 一个请求直接批量传到多个账户多个计划

### 3.2 上传策略

每个文件的后台流程如下：

1. 根据所选计划，按 `advertiser_id` 分组
2. 对每个 `advertiser_id`：
   - 先检查该文件是否已在本账户素材库存在
   - 存在则复用已有 `material_id`
   - 不存在则上传视频素材
3. 拿到该账户的 `material_id` 后：
   - 对该账户下被勾选的每个计划逐条添加素材
4. 记录：
   - 上传成功 / 失败
   - 绑定成功 / 失败
   - 错误信息

### 3.3 去重原则

V1 必须实现同账户内去重。

标准做法：

- 对每个本地文件计算 `sha256`
- 维护：
  - `advertiser_id + file_hash -> material_id`

这样同一个视频文件在同一账户下重复上传时：

- 不再重新传一次
- 直接复用已有 `material_id`

### 3.4 异步执行原则

上传必须走 Celery，不允许走 Web 进程直接上传。

原因：

- 文件数可能到 `50`
- 目标计划可能很多
- 上传和计划绑定总耗时不可控
- 必须防止反向代理超时

因此：

- `POST /api/upload-jobs` 只创建任务并立即返回 `202`
- 真正执行由 Celery Worker 负责

## 4. 技术架构调整

在现有架构上新增一个“素材上传任务域”。

当前正式架构保持不变：

- `web`: FastAPI
- `postgres`: PostgreSQL
- `redis`: Redis
- `worker`: Celery Worker
- `scheduler`: Celery Beat

新增能力落点：

- `web`
  - 上传页面
  - 目标搜索接口
  - 创建上传任务接口
  - 查询任务进度接口
- `worker`
  - 文件预处理
  - 官方素材上传
  - 计划绑定
  - 重试
- `postgres`
  - 上传任务表
  - 文件表
  - 目标计划表
  - 账户素材映射表
- `redis`
  - 任务进度缓存
  - 同账户上传锁

### 4.1 不变的架构原则

1. 页面查询不直接打官方接口
2. 页面只读本地库和任务进度
3. 文件上传动作只能由后台任务执行
4. 官方接口错误不允许阻塞整个页面

## 6. 后端接口设计

### 6.1 目标搜索

`GET /api/upload-targets`

参数：

- `scope=account|plan`
- `keyword=...`

返回：

- `accounts`
- `plans`
- 若 `scope=account`，同时返回这些账户下的计划列表

### 6.2 创建上传任务

`POST /api/upload-jobs`

请求：

- `scope`
- `keyword`
- `selected_account_ids`
- `selected_plan_ids`
- 文件列表

返回：

- `job_id`
- `status=queued`

### 6.3 查询任务进度

`GET /api/upload-jobs/{job_id}`

返回：

- 任务状态
- 文件总数
- 已上传数
- 目标计划总数
- 已绑定计划数
- 成功 / 失败数
- 最近错误

### 6.4 上传任务列表

`GET /api/upload-jobs`

返回：

- 最近任务列表
- 支持按状态筛选

### 6.5 上传失败重试

`POST /api/upload-jobs/{job_id}/retry`

行为：

- 只重试失败文件或失败绑定目标

## 7. Worker 任务拆分

建议拆成 3 段，不要一个大函数到底。

### 7.1 `upload.prepare_job`

职责：

- 校验文件
- 计算 hash
- 计算目标计划
- 写任务主表

### 7.2 `upload.process_file`

职责：

- 对单个文件按账户分组上传
- 复用同账户已存在素材
- 得到账户侧 `material_id`

### 7.3 `upload.bind_to_plans`

职责：

- 将已上传成功的素材逐计划绑定
- 更新每个计划目标状态

### 7.4 并发建议

V1 建议低并发：

- 同账户素材上传：`2`
- 计划绑定：`3~5`

不要无限并发，否则很容易被官方限流或触发不稳定错误。

## 9. 交互细节要求

1. 如果当前范围是 `账户`
   - 页面必须明确告诉用户：最终会上传到这些账户下选中的计划

2. 如果一个账户没有计划
   - 不允许加入上传目标
   - 需要在结果里明确提示

3. 如果用户没有选择计划
   - 不允许创建任务

4. 如果用户没有选文件
   - 不允许创建任务

5. 如果一个文件上传成功但绑定部分失败
   - 任务状态必须是 `partial_success`

6. 前端必须能看到失败原因
   - 不能只显示“失败”

## 10. 与现有归属规则的关系

上传页和归属规则页都允许搜关键词，但业务不同：

- 归属规则：
  - 关键词用于“把对象归到某个归属人”
- 素材上传：
  - 关键词用于“筛选上传目标”

两者可复用搜索实现，但：

- 不共享表单
- 不共享保存逻辑
- 不共享数据库表

## 11. 风险与约束

### 11.1 官方接口约束

当前系统已验证：

- 可拉计划素材
- 可拉计划详情
- 可拉素材表现

但“批量上传素材并绑定计划”的正式实现，仍需在开发阶段再确认这组公开接口的最终可用性与权限：

- 视频上传
- 图片上传
- 素材绑定
- 计划加素材

当前工程上允许先按这些接口能力位设计：

- `POST /open_api/2/file/video/ad/`
- `POST /open_api/2/file/image/ad/`
- `POST /open_api/2/file/material/bind/`
- `POST /open_api/v1.0/qianchuan/uni_promotion/ad/material/add/`

如果联调时发现其中某条接口权限受限，处理原则是：

1. 不推翻上传页和任务架构
2. 只调整上传适配层
3. 数据表和任务流保持不变

### 11.2 本地文件存储

V1 建议：

- 文件先存本地卷
- 由 worker 读取后上传

V2 可切：

- MinIO / S3

## 13. 当前默认约定

若没有额外说明，V1 默认按以下规则开发：

1. 只支持视频上传
2. 目标对象最终只允许是计划
