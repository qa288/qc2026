# 12 初始化配置模板

版本：`v2.1`
状态：`生效中`
最后更新：`2026-03-30`
适用范围：标准部署初始化、环境变量准备、健康检查约定
读者对象：后端、运维、值班同事

> 当前现网服务器已安装 `1Panel`，推荐用它管理站点反代、证书和容器观察；应用编排真源仍然是 `docker-compose.dashboard.yml`。

## 1. 当前配置真源

标准部署的配置来源固定为：

1. `.env.dashboard`
2. `docker-compose.dashboard.yml`
3. 代码默认值

说明：

- `config.json` 仍被运行时代码兼容读取，但在标准部署中只作为可选覆盖项，不再作为默认初始化模板。
- 任何生产配置示例都以 [docker-compose.dashboard.yml](../../server_payload/qianchuan_openclaw_reporter/docker-compose.dashboard.yml) 和 [dashboard/settings.py](../../server_payload/qianchuan_openclaw_reporter/dashboard/settings.py) 为准。

## 2. `.env.dashboard` 模板

参考样例以 [`.env.dashboard.example`](../../server_payload/qianchuan_openclaw_reporter/.env.dashboard.example) 为准。最小可运行模板如下：

```env
DASHBOARD_USERNAME=admin
DASHBOARD_PASSWORD=REPLACE_ME
SESSION_SECRET=REPLACE_ME

POSTGRES_DB=qianchuan
POSTGRES_USER=qianchuan
POSTGRES_PASSWORD=REPLACE_ME

APP_ID=REPLACE_ME
APP_SECRET=REPLACE_ME
REFRESH_TOKEN=REPLACE_ME
CUSTOMER_CENTER_ID=REPLACE_ME

TIMEZONE=Asia/Shanghai
DETAIL_SYNC_INTERVAL_MINUTES=10
EXTENDED_RETENTION_DAYS=35
EXTENDED_HISTORY_REFRESH_DAYS=35
```

常用可选项：

- `FEISHU_TARGET`
- `PLAN_MARKETING_GOALS`
- `PLAN_PAGE_SIZE`
- `OCEANENGINE_TOKEN_LOCK_KEY`
- `OCEANENGINE_TOKEN_LOCK_TIMEOUT_SECONDS`
- `OCEANENGINE_TOKEN_LOCK_BLOCKING_TIMEOUT_SECONDS`

## 3. 标准编排文件

标准编排文件固定为 [docker-compose.dashboard.yml](../../server_payload/qianchuan_openclaw_reporter/docker-compose.dashboard.yml)，不要再维护平行的 `docker-compose.yml` 示例。

当前标准服务：

- `web`
- `worker`
- `scheduler`
- `postgres`
- `redis`

关键约定：

- `env_file` 统一使用 `.env.dashboard`
- Worker 启动命令：`celery -A dashboard.celery_app.celery_app worker -l info`
- Scheduler 启动命令：`celery -A dashboard.celery_app.celery_app beat -l info`
- 数据目录映射：`./data:/app/data`
- PostgreSQL 数据目录：`./volumes/postgres`
- Redis 数据目录：`./volumes/redis`

标准启动命令：

```bash
set -a && . ./.env.dashboard && set +a
docker-compose -f docker-compose.dashboard.yml up -d --build
```

## 4. 1Panel 反向代理建议

在 `1Panel` 中建议配置：

- 站点域名：`qc.tyos.cc`
- 反向代理目标：`127.0.0.1:9898`
- 开启 HTTPS
- 保留 `Host`、`X-Forwarded-For`、`X-Forwarded-Proto`

补充：

- `1Panel` 负责反代、证书和容器观察
- Compose 文件才是应用编排真源
- 不允许只在 `1Panel` 页面手工改容器参数而不回写 Compose

## 5. 健康检查约定

`GET /healthz` 只用于进程存活检查，标准返回：

```json
{"status":"ok"}
```

`GET /readyz` 用于检查数据库、Redis、Celery 和 schema 版本，标准返回结构示例：

```json
{
  "status": "ok",
  "ok": true,
  "checks": {
    "database": {
      "ok": true,
      "backend": "postgresql",
      "schema_version": 1,
      "expected_schema_version": 1
    },
    "redis": {"ok": true, "status": "ok"},
    "celery_broker": {"ok": true, "status": "ok"},
    "celery_result_backend": {"ok": true, "status": "ok"}
  }
}
```
