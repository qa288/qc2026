# DOCS 文档导航

版本：`v2.1`
状态：`生效中`
最后更新：`2026-03-30`
适用范围：`DOCS/` 全目录
读者对象：研发、运维、联调同事

本项目所有正式文档统一维护在 `DOCS/`。
文档引用统一使用 `DOCS/` 标准写法，禁止混用小写 `docs/`，避免在大小写不敏感文件系统上产生歧义。

## 目录架构

- `standard/`
  - 正式标准、决策、技术基线、运维基线、计划排期
- `runtime/`
  - 当前线上实现、运行事实、接口清单、运行约束
- `runtime/capabilities/`
  - 官方接口能力发现、验证结果、发现手册
- `integrations/`
  - 单专题集成与背景设计
- `prompts/`
  - 提示词、代理约束和辅助材料

## 使用规则

- 任何新文档都必须归类到 `DOCS/` 现有分层，禁止再新增平行文档根目录。
- 文档内统一使用相对路径，禁止写本机绝对路径如 `/Users/xy/...`。
- 同类信息只能保留一个专题真源，其他文档只做摘要或引用。
- 代码、部署、运维调整后，必须同步更新相应真源文档。
