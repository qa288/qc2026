## 2. 当前系统目标

系统面向老板、运营主管和系统维护者，解决这 4 类问题：

- 实时看整体经营
- 实时看账户、计划、商品、员工、素材表现
- 基于阈值提醒异常
- 在官方接口异常时保持可用降级

## 3. 当前技术实现

### 3.1 页面与接口

- 页面服务：`FastAPI`
- 前端：`Jinja2 + 原生 JS/CSS`
- 访问端口：`9898`
- 访问方式：公网访问 + 登录保护

### 3.2 后端运行角色

- `web`
- `worker`
- `scheduler`
- `postgres`
- `redis`

### 3.3 数据存储

- PostgreSQL：结构化数据、规则、事件、token 真源
- `/app/data`：token 缓存副本、导出文件和必要运行数据

### 3.4 查询边界

当前正式约束：

- 页面查询接口只读 PostgreSQL / 内存缓存
- 页面查询不允许直接调用 OceanEngine 远程报表
- 远程官方接口只允许用于：
  - 定时同步
  - 历史缺口回补
  - 手动补数
  - token 换取与刷新

## 4. 不可回退的系统约束

- 只能使用巨量官方 `open_api`
- 不允许改回浏览器抓取、Cookie、`/nbs/api/*`
- 必须保留登录保护
- 必须保留根路径公开归属人榜
- 必须保留分钟级历史快照
- 必须保留细粒度扩展快照
- 必须保留账户、计划、商品、员工、素材五类可视数据
- 必须保留阈值告警规则与事件
- 必须保留排序、筛选、指定日期范围
- 必须保留计划状态中文显示

## 6. 数据刷新与调度

当前已实现：

- 主快照同步：每 `1` 分钟
- 细粒度同步：默认每 `10` 分钟
- 阈值告警发送：每 `1` 分钟
- 浏览器轮询：每 `60` 秒
- 手工同步改为 Celery 入队，接口立即返回 `202`

## 6.1 历史回补

当前已实现：

- 查询 `昨日 / 近7天 / 近30天 / 指定日期范围` 时，会先检查本地是否缺失历史日快照
- 如果缺失，则按天调用官方接口回补后再落库
- 回补完成后，后续同范围查询继续只读库

说明：

- 第一次命中缺历史数据的范围时，请求可能明显慢于平时
- 一旦回补完成，同范围查询恢复到读库速度

### 7.4 素材榜性能口径

当前已实现：

- 细粒度同步会先写原始素材明细：
  - `material_snapshots`
  - `video_origin_flags`
- 再写素材预聚合：
  - `material_rollups`
- 素材榜查询优先走 `material_rollups`
- 只有历史老快照还没有 rollup 时，才回退到 `material_snapshots` 临时聚合
