## 2. 当前系统目标

系统面向老板、运营主管和系统维护者，解决这 4 类问题：

- 实时看整体经营
- 实时看账户、计划、商品、员工、素材表现
- 基于阈值提醒异常
- 在官方接口异常时保持可用降级

## 3. 当前技术实现

### 3.1 页面与接口

- 页面服务：`FastAPI`
- 前端：`Jinja2 + 原生 JS/CSS`
- 访问端口：`9898`
- 访问方式：公网访问 + 登录保护

### 3.2 后端运行角色

- `web`
- `worker`
- `scheduler`
- `postgres`
- `redis`

### 3.3 数据存储

- PostgreSQL：结构化数据、规则、事件、token 真源
- `/app/data`：token 缓存副本、导出文件和必要运行数据

## 4. 不可回退的系统约束

- 只能使用巨量官方 `open_api`
- 不允许改回浏览器抓取、Cookie、`/nbs/api/*`
- 必须保留登录保护
- 必须保留分钟级历史快照
- 必须保留细粒度扩展快照
- 必须保留账户、计划、商品、员工、素材五类可视数据
- 必须保留阈值告警规则与事件
- 必须保留排序、筛选、自定义时间段
- 必须保留计划状态中文显示

## 6. 数据刷新与调度

当前已实现：

- 主快照同步：每 `1` 分钟
- 细粒度同步：默认每 `10` 分钟
- 阈值告警发送：每 `1` 分钟
- 浏览器轮询：每 `60` 秒
