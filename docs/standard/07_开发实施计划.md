# 07 开发实施计划

## 1. 目标

本计划用于指导从标准文档到 V1 可上线版本的实施顺序。  
每一阶段必须可运行、可验证、可交付。

---

## 2. 实施原则

1. 先统一架构，再开始编码。
2. 先把数据链路跑通，再做页面细节。
3. 先把同步、聚合、告警做稳，再做扩展功能。
4. 先满足内部使用，再考虑公开页、大屏、日报等二期项。

---

## 3. 阶段划分

### 阶段 0：基础骨架

任务：

- 搭建 FastAPI + PostgreSQL + Redis + Celery + Docker Compose
- 建立 Web / Worker / Scheduler 服务
- 配置基础 env、日志、健康检查

完成标准：

- `docker compose up` 后核心服务可启动
- Web、PostgreSQL、Redis、Worker、Scheduler 健康检查可访问

### 阶段 1：认证与 Token 基础

任务：

- 登录、退出、当前用户
- 基础角色与审计
- Token 单点刷新
- 最新 token 数据库存储与缓存文件
- 提供 token-latest 内部接口

完成标准：

- 登录可用
- token 刷新链路可用
- 其它服务可读取最新 token

### 阶段 2：官方集成与结构同步

任务：

- 子账户列表
- 计划列表
- 计划详情
- 商品
- 素材
- 视频相关明细

完成标准：

- 能稳定落库一批测试账户
- 同步任务失败可记录

### 阶段 3：报表与聚合

任务：

- 账户、计划、商品、素材快照
- 账户汇总接口降级
- 榜单聚合与缓存

完成标准：

- 页面不依赖实时打官方接口
- 账户汇总异常时仍可展示计划聚合结果

### 阶段 4：页面与交互

任务：

- 登录页
- 主看板
- 账户页
- 经营拆解页
- 计划页
- 素材页
- 通知规则页

完成标准：

- 页面流程闭环可用
- 排序、搜索、时间段、自定义区间可用

### 阶段 5：阈值告警

任务：

- 规则管理
- 告警评估
- OpenClaw 通知
- 告警日志

完成标准：

- 阈值规则可配置
- 命中后可通知
- 去重和恢复可追踪

### 阶段 6：运维与上线

任务：

- 持久化卷
- 备份
- 监控与健康检查
- 上线与回滚

完成标准：

- 单机环境稳定运行
- 运维可追踪同步、告警、Token 和日志
