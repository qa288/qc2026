# 13 已确认决策基线

最后更新：`2026-03-22`

后续开发、联调、部署如与本文档冲突，以本文档为准。

## 1. 产品与路由

- 产品定位：内部经营与投流决策平台，带一个匿名可访问的归属人排名公开页
- 路由：
  - `/`
  - `/login`
  - `/workbench`
  - `/api/*`
- 第一阶段做公开页，不做大屏页、独立 `/admin`

## 2. 部署与域名

- 正式域名：`qc.tyos.cc`
- 部署方式：Linux 单机 + Docker Compose
- 服务器已安装 `1Panel`，作为反向代理、站点管理和 Docker 运维入口
- 公开页可匿名访问，后台可公网开放，但保留登录

## 3. 技术栈

- Web/API：FastAPI
- 主数据库：PostgreSQL
- 缓存 / 队列 / 锁：Redis
- 异步任务：Celery
- 定时调度：独立 scheduler
- 前端：轻前端

## 4. 通知

- 第一阶段只做阈值告警
- 先完成渠道配置结构与适配层
- 实际手机通知和定时通知可后置开启
- 预留 Feishu / DingTalk / Email / Slack / Telegram

## 5. 账户汇总

- 允许官方账户汇总失败时自动降级为计划聚合
- 页面和导出必须保留来源标记

## 5A. 权限与归属

- 第一阶段开始建设角色与账户授权模型
- 角色至少区分：匿名公开、运营、管理员
- 员工归属按“关键词命中 + 人工修正”全面建设
- 排名统计采用单归属口径，搜索结果允许多命中

## 6. Token

- 单点刷新，共享最新 token
- PostgreSQL 为主真源
- `/app/data/qianchuan_latest_token.json` 为缓存副本
- Redis 负责刷新锁
- 其它服务器只读，不自行刷新
