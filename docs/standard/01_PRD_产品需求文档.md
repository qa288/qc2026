# 01 PRD - 千川投放趋势大盘

## 1. 产品定位

这是一个 **公司内部使用** 的千川经营与投流决策平台。  
目标不是替代官方投放控制台，而是在官方开放接口之上完成：

- 公开归属人排名页
- 多账户统一采集
- 账户 / 计划 / 商品 / 素材 / 员工多维排名
- 历史趋势与时间段对比
- 阈值告警
- 内部运营查看与决策支持

---

## 2. 项目目标

### 2.1 核心目标

- 支撑约 `1000` 个账户规模的数据采集和展示
- 统一查看账户、计划、商品、素材、员工表现
- 支持分钟级同步和历史保留
- 支持指定日期范围查询
- 支持阈值告警并通过 OpenClaw 发送通知

### 2.2 第一阶段目标

第一阶段只解决最关键的问题：

- 打开网页直接看公开归属人排名
- 看整体经营概况
- 看账户表现
- 看计划表现
- 看商品/归属人排名
- 看素材表现
- 做角色和账户授权
- 做关键词归属和人工修正
- 设置阈值告警
- 保证 Token、同步、降级和运维链路稳定

### 2.3 第一阶段非目标

第一阶段不做：

- 大屏模式
- 定时简报
- 浏览器抓取
- 多租户 SaaS
- 单点登录
- 复杂 BI 设计器

---

## 3. 用户角色

### 3.1 管理员

- 可查看全部数据
- 可配置通知和规则
- 可查看系统状态、同步状态、Token 状态
- 可维护归属人、关键词、人工修正与账户授权

### 3.2 运营

- 需要登录
- 在授权范围内查看经营数据
- 可查看账户、计划、素材、商品、员工相关明细
- 与管理员使用同一工作台，但仅可见授权账户范围

### 3.3 匿名公开访问者

- 无需登录
- 只能查看公开归属人排名页
- 不可查看后台配置、系统状态、余额、钱包、Token 等敏感信息

### 3.4 老板 / 主管

- 需要登录
- 查看整体经营结果、重点异常、各维度排名
- 不要求复杂配置能力

---

## 4. 核心业务规则

### 4.1 数据来源

- 只接千川官方开放接口
- 不允许以前端或脚本抓官方后台页面作为主方案

### 4.2 账户汇总降级

- 优先使用官方账户汇总接口
- 若官方账户汇总接口返回 `50000` 或持续不可用
- 自动降级为计划聚合账户汇总
- 页面必须标记数据来源

### 4.3 Token 规则

- 只允许一个实例刷新 Token
- 其它实例只读最新 Token
- Token 必须滚动保存最新 `refresh_token`

### 4.4 通知规则

- 第一阶段只做阈值告警
- 第一阶段先做规则引擎、记录和渠道配置结构
- 实际发送可后置按开关启用

---

## 5. 页面范围

### 5.1 登录页

- 账号密码登录
- 登录后进入内部工作台

### 5.2 公开归属人排名页

- 匿名可访问
- 默认按消耗降序
- 支持排序方向和时间范围切换
- 不展开计划和素材细节

### 5.3 主看板

包含以下主模块：

- 总览
- 账户
- 经营拆解
- 计划
- 素材
- 员工归属
- 账户授权
- 通知规则

### 5.4 交互要求

- 所有榜单支持排序
- 自动刷新后保留当前排序方式
- 原本按 `日 / 周 / 月` 的模块统一支持 `日 / 昨日 / 周 / 月 / 指定日期范围`
- 详情统一在当前页面内展开，不跳多层页面
- 员工归属支持关键词搜索命中 + 人工勾选修正

---

## 6. 告警范围

第一阶段仅支持阈值告警：

- 通用余额预警
- 共享钱包余额预警
- 消耗阈值预警
- 爆单预警
- 失败重试与去重

不做：

- 定时简报
- 日报 / 周报
- 多通道联动策略

---

## 7. 成功标准

第一阶段完成的判断标准：

- 能稳定采集并展示多账户数据
- 账户 / 计划 / 商品 / 素材 / 员工数据可按时间段查询
- 官方账户汇总异常时，页面仍能正常展示计划聚合结果
- 告警规则可配置、可触发、可通知
- 登录、同步、日志、Token 链路稳定
