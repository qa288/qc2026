# 11 字段字典与排序规则

## 1. 设计原则

1. 数据库尽量多存，前台按页面需求少展示。
2. 字段显示、排序、筛选都应配置化。
3. 查不到的数据返回 `null`，页面展示 `--`。
4. 第一阶段先围绕“消耗”完成核心体验。

---

## 2. 主看板核心字段

## 2.1 默认展示字段
| 字段 | 说明 | 是否排序 | 是否搜索 |
|---|---|---:|---:|
| rank | 排名 | 否 | 否 |
| name | 当前维度名称 | 否 | 是 |
| spend | 消耗金额 | 是（固定降序） | 否 |
| updatedAt | 更新时间 | 否 | 否 |

## 2.2 预留字段
| 字段 | 说明 |
|---|---|
| payOrders | 支付订单量 |
| payAmount | 成交金额 |
| roiPay | 支付 ROI |
| campaignCount | 计划数 |
| materialCount | 素材数 |

---

## 3. 后台统一排名页字段

## 3.1 推荐第一阶段默认展示字段
| 字段 | 说明 | 可排序 |
|---|---|---:|
| name | 当前维度名称 | 否 |
| spend | 消耗金额 | 是 |
| payAmount | 整体成交金额 | 是 |
| netPayAmount | 净成交金额 | 是 |
| roiPay | 整体支付 ROI | 是 |
| roiNet | 净成交 ROI | 是 |
| payOrders | 支付订单量 | 是 |
| netPayOrders | 净成交订单量 | 是 |
| updatedAt | 更新时间 | 否 |

## 3.2 可扩展字段
| 字段 | 说明 |
|---|---|
| accountBalance | 账户余额 |
| walletValidBalance | 共享钱包可用余额 |
| campaignCount | 计划数 |
| materialCount | 素材数 |
| refundRate1h | 1 小时付前退货率 |
| settlementRateNet | 净成交结算率 |

## 6. 排序规则

## 6.1 主看板默认
- 字段固定：`spend`
- 顺序固定：`desc`

## 6.2 后台
### 默认排序
- 字段：`spend`
- 顺序：`desc`

### 可选排序字段
- `spend`
- `payAmount`
- `netPayAmount`
- `roiPay`
- `roiNet`
- `payOrders`
- `netPayOrders`
- 后续扩展字段

### 空值处理
- `null` 排在最后
- 前端显示 `--`

## 11. 页面字段配置建议

建议维护一份配置文件：

```ts
export const rankingFieldConfig = {
  default: ["rank", "name", "spend", "updatedAt"],
  dashboardDefault: [
    "name",
    "spend",
    "payAmount",
    "netPayAmount",
    "roiPay",
    "payOrders",
    "updatedAt"
  ],
  sortable: [
    "spend",
    "payAmount",
    "netPayAmount",
    "roiPay",
    "roiNet",
    "payOrders",
    "netPayOrders"
  ]
};
```

这样后续增加字段时，不需要改动大量页面逻辑。
